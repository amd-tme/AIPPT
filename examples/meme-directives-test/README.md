# Example: Meme Directives Test

This example shows a real deck generated by the `/create-deck` skill using the **pptxgenjs** engine. The source outline (`outline-with-directives.md`) deliberately exercises every `LAYOUT:` and `IMAGE:` directive combination, making this the best reference for how directives affect slide rendering.

## Files

| File | Description |
|------|-------------|
| `outline-with-directives.md` | The source outline — demonstrates `LAYOUT:` and `IMAGE:` directives |
| `meme-directives-test.mjs` | The generated build script (ES Modules, pptxgenjs) |
| `meme-directives-test.pptx` | The generated presentation |
| `meme-directives-test.pdf` | PDF export for quick preview |

## Running the script

From the repo root:

```bash
npm install -g pptxgenjs react react-dom react-dom-server react-icons sharp
node examples/meme-directives-test/meme-directives-test.mjs
```

The script writes `output/meme-directives-test.pptx`. It depends on `lib/pptxgenjs-helpers.mjs` and `themes/default.yaml`, both in the repo root.

> **Note:** Some slides reference meme images from `outlines/memes/`. Those images are not tracked in the repo — slides with missing images render a placeholder rectangle instead.

## What this example shows

### Directives in the outline

`LAYOUT:` and `IMAGE:` are optional lines written directly in a slide's content block, before the bullets. The skill strips them before rendering — they only affect how the slide is built.

```markdown
# Stages of Debugging
LAYOUT: numbered
- Denial: "Works on my machine"
- Blame: The compiler, the OS, or cosmic rays
...
```

### Directive combinations across the deck

| Slide | Directive(s) | Layout used |
|-------|-------------|-------------|
| Title | *(none)* | Title |
| This Is Fine | `IMAGE:` | Image + bullets side-by-side |
| Two Types of Developers | `LAYOUT: two_column` | Two-column comparison |
| Stages of Debugging | `LAYOUT: numbered` | Process flow (6 steps) |
| Distracted Boyfriend | `IMAGE:` | Image + bullets side-by-side |
| Drake Hotline Bling | `LAYOUT: two_column` | Two-column comparison |
| One Does Not Simply | `IMAGE:` | Image + bullets side-by-side |
| Expanding Brain | `LAYOUT: numbered` | Process flow (4 steps) |
| Change My Mind | `IMAGE:` | Image + bullets side-by-side |
| Is This a Pigeon? | `IMAGE:` | Image + bullets side-by-side |
| Always Has Been | `LAYOUT: two_column` | Two-column comparison |
| Closing | *(none)* | Closing |

### Module format

This script uses ES Modules (`import`/`export`) and the `.mjs` extension — the format used by all skill-generated scripts after the shared helper library (`lib/pptxgenjs-helpers.mjs`) was introduced. Contrast with `examples/slides-as-code-design/slides-as-code-design.js`, which is self-contained CommonJS (`require()`).
