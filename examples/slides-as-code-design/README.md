# Example: Slides as Code Design

This example shows a real deck generated by the `/create-deck` skill using the **pptxgenjs** engine. The deck explains the "slides as code" design concept behind AIPPT's conversational editing workflow.

## Files

| File | Description |
|------|-------------|
| `slides-as-code-design.js` | The build script — this is the source of truth for the deck |
| `slides-as-code-design.pptx` | The generated presentation |
| `slides-as-code-design.pdf` | PDF export for quick preview |

## Running the script

To regenerate the PPTX from the source script, install the required npm packages and run from the repo root:

```bash
npm install -g pptxgenjs react react-dom react-dom-server react-icons sharp
node examples/slides-as-code-design/slides-as-code-design.js
```

The script writes `output/slides-as-code-design.pptx`.

## What this example shows

- **Theme constants and layout helpers** — colors, fonts, and safe-area geometry defined at the top; reusable layout functions below
- **Layout variety** — title, section divider, bullet, icon rows, card grid, three-column cards, two-column, process flow, and code block slides
- **Icon rendering** — `react-icons` rendered to SVG via React, then rasterized to PNG with `sharp` for embedding in pptxgenjs
- **Speaker notes with metadata** — the `addBulletSlide` function accepts a `notes` argument; slide 5 demonstrates the `---aippt-meta---` format
- **CommonJS module format** — uses `require()` rather than `import`; the `.js` extension is intentional

## Note on `.js` vs `.mjs`

Scripts generated by the pptxgenjs engine come in two formats depending on when they were created:

- **`.js`** — CommonJS (`require()`), compatible with Node.js default module resolution
- **`.mjs`** — ES Modules (`import`), used by newer skill-generated scripts

Both formats are valid. Do not rename a `.js` file to `.mjs` without converting all `require()` calls to `import` statements first.
