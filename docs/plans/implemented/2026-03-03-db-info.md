# PRD: Database Info Dump (`db-info`)

**Date:** 2026-03-03
**Author:** Matt
**Status:** Draft

---

## Summary

Add a `db-info` CLI subcommand that dumps a comprehensive snapshot of the SQLite catalog database -- schema DDL, table statistics, deck inventory, per-slide detail, tags, taxonomy, and edit history. Supports plain-text (default) and JSON output. Useful as a diagnostic tool before and after major code changes.

## Motivation

- **What problem does this solve?** When making schema migrations, re-ingesting decks, or debugging catalog issues, there's no quick way to see the full state of the database. Currently you have to run ad-hoc SQL queries or inspect the web UI piece by piece.
- **Who benefits?** Developer (primary). Useful for documenting state before/after PRs that touch the data model.
- **What happens if we don't do this?** Manual SQL queries continue to work but are tedious and inconsistent across sessions.

## Requirements

### Must Have

- [ ] New `db-info` CLI subcommand
- [ ] Plain-text output (default) with human-readable formatting
- [ ] JSON output via `--json` flag
- [ ] Database metadata: file path, file size, SQLite version
- [ ] Schema DDL from `sqlite_master` (CREATE TABLE + CREATE INDEX statements)
- [ ] Row counts for all tables
- [ ] Full deck inventory with metadata (name, path, hash, author, dates, slide count, sections)
- [ ] Per-slide detail per deck: position, title, content_hash, layout_type, tags, section, notes (truncated)
- [ ] Tag summary with source and usage counts
- [ ] Taxonomy entries grouped by category
- [ ] Edit history summary with recent entries
- [ ] Optional `--output` flag to write to a file instead of stdout

### Nice to Have

- [ ] `--compact` flag to show summary only (skip per-slide detail)

### Out of Scope

- Database writes or modifications
- Web UI integration (the web UI already shows most of this per-deck)
- Diffing two database snapshots (users can diff the text/JSON output themselves)

---

## Design

### Approach

New read-only module `outline2ppt/dbinfo.py` with a `dump_db_info()` function that queries the database and returns a formatted string. The CLI handler `cmd_db_info()` calls this function and prints or writes the result. Follows the existing pattern of `cmd_export`, `cmd_tags`, etc.

### New/Modified Modules

| Module | Change | Description |
|--------|--------|-------------|
| `outline2ppt/dbinfo.py` | New | Core logic: query DB, format output as text or JSON |
| `outline2ppt/cli.py` | Modified | Add `db-info` subcommand parser and `cmd_db_info()` handler |

### Data Model Changes

No data model changes. This is a read-only diagnostic command.

---

## CLI Changes

### New Commands

```
outline2ppt.py db-info [--db slides.db] [--json] [--output file.txt]
```

| Argument | Type | Default | Description |
|----------|------|---------|-------------|
| `--db` | str | `slides.db` | Path to the SQLite database |
| `--json` | flag | off | Output as JSON instead of plain text |
| `--output` | str | stdout | Write output to a file |

### Example Usage

```bash
# Default: plain text to stdout
python outline2ppt.py db-info

# JSON output
python outline2ppt.py db-info --json

# Save snapshot before a migration
python outline2ppt.py db-info --output snapshots/pre-migration.txt

# Different database
python outline2ppt.py db-info --db other.db --json --output other-db.json
```

## Risks & Open Questions

- **Risk:** Large databases could produce very long output. Mitigated by notes truncation (80 chars) and future `--compact` flag.
- **Risk:** `sqlite_master` DDL may differ from `schema.sql` if migrations added columns via ALTER TABLE. This is expected and useful -- shows the actual runtime schema.
