# PRD: Enhanced Initial Generation

**Date:** 2026-03-02
**Author:** Matt
**Status:** Draft

---

## Summary

Upgrade `create --enhance` to produce higher-quality first-pass output by adding font sizing, bold lead-in formatting, column headers for two-column layouts, numbered list support for sequential content, and placeholder image shapes for diagram suggestions. These changes address the most common and repetitive feedback from the multimodal analysis pipeline.

## Motivation

- **What problem does this solve?** Every slide generated by `--enhance` receives the same feedback: text too small, whitespace waste, no visual hierarchy, two-column splits lack headers. These are mechanical formatting issues, not content problems.
- **Who benefits?** End users get presentable decks on the first run instead of needing manual fixes. The `improve` command (PRD: Improve Command) benefits from a better baseline.
- **What happens if we don't do this?** Users must manually fix font sizes and formatting on every slide, or the `improve` command wastes LLM tokens on obvious fixes.

## Requirements

### Must Have

- [ ] Body text font size set to Pt(22) for level 0-1 bullets, Pt(18) for level 2
- [ ] Bold lead-in words: when a bullet starts with a keyword followed by `:` or ` — `, bold the lead-in portion
- [ ] Column headers for two-column layouts: LLM provides header labels in LAYOUT response, applied as bold first paragraphs in each column placeholder
- [ ] `numbered` layout type: enhance prompt can select it for sequential content; renders `1.`, `2.`, `3.` prefixes instead of bullets, uses "Title and Content" slide layout
- [ ] Placeholder image shapes: when LLM selects `diagram` but image gen is disabled, insert a light gray rectangle with descriptive text instead of silent fallback to bullet

### Nice to Have

- [ ] Subtitle/tagline support for title slides (basic layout): LLM can provide a one-line value proposition rendered below the title
- [ ] Automatic font size scaling when content is sparse (fewer than 3 bullets → larger font)

### Out of Scope

- Changing template fonts, colors, or backgrounds (template-controlled)
- Adding icons, real images, or complex shapes
- Web UI changes
- The `improve` command (separate PRD)

## Design

### Approach

Modify the existing layout application functions in `layouts.py` and the enhance prompt in `enhancer.py`. All changes are additive — existing behavior is preserved for non-enhanced creation.

### New/Modified Modules

| Module | Change | Description |
|--------|--------|-------------|
| `outline2ppt/layouts.py` | Modified | `_apply_bullets_to_text_frame()` gains font sizing and bold lead-in formatting. New `apply_numbered_layout()` function. New `apply_placeholder_image()` function. |
| `outline2ppt/enhancer.py` | Modified | Enhancement prompt updated to include `numbered` layout option and column header output format. LAYOUT response format extended to `LAYOUT: two_column \| Left Header \| Right Header`. |
| `outline2ppt/parser.py` | Modified | `parse_llm_suggestions()` updated to parse column headers from LAYOUT line. |
| `outline2ppt/cli.py` | Modified | `_add_slide()` updated to pass column headers to `apply_two_column_layout()` and handle `numbered` layout type. Diagram fallback creates placeholder image instead of silent bullet conversion. |

### Data Model Changes

No data model changes.

## Risks & Open Questions

- **Risk:** Bold lead-in detection is heuristic (split on `:` or ` — `). May bold unintended content. Mitigation: only apply when the lead-in is 1-4 words.
- **Risk:** LLM may not consistently output column headers in the expected format. Mitigation: graceful fallback — if no headers parsed, skip them (current behavior).
- **Question:** Should `numbered` be a separate layout type or a modifier on `bullet`? Decision: separate type is cleaner for prompt and code clarity.
- **Question:** Should placeholder images have a fixed size or scale to available space? Decision: fixed size (centered, ~60% of content area) for v1.
